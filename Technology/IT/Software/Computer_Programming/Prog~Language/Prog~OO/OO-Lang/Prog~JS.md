## #has_/text_of_/abstract 

> **JavaScript**, often abbreviated as JS, is a programming language 
> and core technology of the World Wide Web, alongside HTML and CSS. 
> 
> As of 2023, 98.7% of websites use JavaScript on the client side 
> for webpage behavior, often incorporating third-party libraries. 
> All major web browsers have a dedicated JavaScript engine to execute the code on users' devices.
>
> JavaScript is a high-level, often just-in-time compiled language that conforms to the ECMAScript standard. 
> It has dynamic typing, prototype-based object-orientation, and first-class functions. 
> It is multi-paradigm, supporting event-driven, functional, and imperative programming styles. 
> It has application programming interfaces (APIs) for working with text, dates, regular expressions, 
> standard data structures, and the Document Object Model (DOM).
> 
> The ECMAScript standard does not include any input/output (I/O), 
> such as networking, storage, or graphics facilities. 
> In practice, the web browser or other runtime system provides JavaScript APIs for I/O.
>
> JavaScript engines were originally used only in web browsers, 
> but are now core components of some servers and a variety of applications. 
> The most popular runtime system for this usage is Node.js.
>
> Although Java and JavaScript are similar in name, syntax, and respective standard libraries, 
> the two languages are distinct and differ greatly in design.
>

## Extended null checks using falsy values 

'falsy' values are `false`, `0`, `""`, `null`, `undefined`, or `NaN`.
They evaluate to `false` when cast to `boolean` and select e.g. the `else` Branch of an `if` statement. 
This makes it quite convenient to check without the need of an Operator. 
Also the `||=` Operator works similar to the C# `??=` Operator.

## How to determine the Data-Shape 

In JavaScript, objects can have various shapes and structures, and there are several ways to examine and understand them:

### Using `console.log(arg)` 
   Displays the object.toString() Result in the console of your browser's developer tools 
   or the console in a Node.js environment. 

### Using `typeof(arg)` Operator 

The `typeof` operator can provide basic information about the type of an object. 
However, it is limited and will only tell you the primitive Type or `object`. 

### Using `Object.keys(arg)` 
   Returns an array of a given object's own enumerable property names. 
   This does not include properties from the object's prototype chain, 
   and it only returns the names of properties that don't have the `{ DontEnum }` attribute. 

### Using `JSON.stringify(arg)` 
Converts a JavaScript object to a JSON string, 
which can help visualize the structure of an object, when formatted. 

### `Object.getOwnPropertyDescriptors(arg)`  
   Returns __all__ own property descriptors of a given object. This provides detailed information about each property, including its value, writable, enumerable, and configurable attributes. For example:

### All Methods and Properties of all base-Prototypes 

```ts
function getAllMethodsAndProperties(obj) {
    let props = {};

    // Iterate over the object and its prototypes
    for(curr = obj;
	    curr; 
	    curr = Object.getPrototypeOf(curr)) {
        let ownProps = Object.getOwnPropertyNames(curr);
        ownProps = ownProps
	        .concat(Object.getOwnPropertySymbols(curr));

        // Add own properties to the result
        ownProps.forEach(prop => {
            props[prop] = Object.getOwnPropertyDescriptor(curr, prop);
        });
    }

    return props;
}

const allMethodsAndProps = getAllMethodsAndProperties(obj);
console.log(allMethodsAndProps);
```

These methods provide different perspectives on the structure of a JavaScript object; 
depending on your needs, you may find one more useful than the others.
